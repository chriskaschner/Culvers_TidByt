<objective>
Fix the Compare page to initialize with a single geolocated store instead of showing the empty state. When a user has exactly 1 store (either saved in custard:compare:stores or inherited from custard-primary), show that store's 3-day schedule in the grid with a placeholder slot hinting they can add more stores for comparison.

Purpose: Currently, the Compare page requires MIN_COMPARE_STORES=2 to render the grid. Users who arrive via geolocation with only 1 store selected see an empty "Add stores" prompt instead of useful flavor data. Lowering the minimum to 1 shows data immediately while still encouraging the comparison workflow.

Output: Updated compare-page.js with 1-store support, plus browser tests.
</objective>

    // Fallback: check legacy primary store key
