<objective>
Fix the Today page so returning users who already have a geolocated store saved in localStorage see the hero card section immediately on load -- never flashing the onboarding banner.

Purpose: Currently, the init() function in today-page.js shows the empty-state section by default, then asynchronously loads the store manifest, then checks localStorage. This causes returning users to briefly see the onboarding banner before it hides and the forecast loads. This plan eliminates that flash by checking localStorage synchronously before any DOM state decisions, showing a loading skeleton for returning users while stores load.

Output: Updated today-page.js with correct init flow, plus browser tests verifying the behavior.
</objective>

1. **Check localStorage synchronously BEFORE the async store load.** Right after `cacheDomRefs()` and `bindEvents()`, read `custard-primary` from localStorage. If a slug exists, immediately:
   - Keep `emptyState.hidden = true` (it starts hidden in HTML)
   - Set `todayLoading.hidden = false` to show loading skeleton
   - Set `todaySection.hidden = true` (will show after data loads)
   This ensures returning users see a loading skeleton, never the onboarding banner.

2. **Move the slug validation into the Promise.all callback** (after manifest loads). Inside the `.then()`:
   - If `savedSlug` exists AND is found in `_allStores`, call `selectStore(savedSlug)`
   - If `savedSlug` exists but is NOT found in `_allStores` (invalid slug), clear it from localStorage (`localStorage.removeItem('custard-primary')`), hide the loading skeleton, and show the empty state as if first visit. Also render quick-start stores.
   - If no `savedSlug` AND no `_currentSlug`, show empty state (first-time visitor)

3. **Keep the existing selectStore() function unchanged** -- it already hides emptyState and calls loadForecast().

The updated init() should look approximately like:

```javascript
function init() {
    cacheDomRefs();
    bindEvents();

    // Synchronous localStorage check -- determine initial DOM state
    // BEFORE any async work to prevent onboarding banner flash
    var savedSlug = null;
    try { savedSlug = localStorage.getItem('custard-primary'); } catch (e) {}

    if (savedSlug) {
      // Returning user: show loading skeleton, keep empty-state hidden
      if (emptyState) emptyState.hidden = true;
      if (todayLoading) todayLoading.hidden = false;
      if (todaySection) todaySection.hidden = true;
      if (weekSection) weekSection.hidden = true;
      if (updatesCta) updatesCta.hidden = true;
    }
    // If no savedSlug, leave DOM as-is (empty-state starts hidden in HTML,
    // will be shown below after manifest loads)

    Promise.all([loadStores(), loadFlavorColors()]).then(function () {
      renderQuickStartStores();

      if (savedSlug) {
        var storeExists = _allStores.find(function (s) { return s.slug === savedSlug; });
        if (storeExists && _currentSlug !== savedSlug) {
          selectStore(savedSlug);
        } else if (!storeExists) {
          // Invalid slug -- clear it and show onboarding
          try { localStorage.removeItem('custard-primary'); } catch (e) {}
          if (todayLoading) todayLoading.hidden = true;
          if (emptyState) emptyState.hidden = false;
          if (todaySection) todaySection.hidden = true;
          if (weekSection) weekSection.hidden = true;
          if (updatesCta) updatesCta.hidden = true;
        }
      } else if (!_currentSlug) {
        if (emptyState) emptyState.hidden = false;
        todaySection.hidden = true;
        if (weekSection) weekSection.hidden = true;
        if (updatesCta) updatesCta.hidden = true;
      }
    });

    // Register service worker
    if ('serviceWorker' in navigator) {
      navigator.serviceWorker.register('sw.js').catch(function () {});
    }
}
```

Do NOT change any other functions (selectStore, loadForecast, etc.) -- the bug is purely in the init() timing. Ensure the existing today-hero.spec.mjs tests still pass after changes.
